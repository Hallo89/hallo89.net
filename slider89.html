#Introduction

If you have at some point needed an input type=range while developing a website you'll know that they are awesome. They fit with a lot of use cases and are very convenient to both interact with as user and deal with as developer.
However, the restriction of not being able to style it very well is a point which isn't easy to ignore. People tell me that there are many ways to style a range input, to which I answer with the question whether approximately 80 lines of css (for cross-browser support) solely for one range input is worth it.

This is why I wrote a framework to solve this very problem: Slider89, which also happened to solve a lot more problems and make life much easier. How convenient!

#What is this?

Slider89 is a very simple framework for easily creating css sliders (range inputs). Being based on javascript and custom elements with CSS instead of a built-in browser-side <input>-element, it features full styling customizability but allows for rich configuration on the technical side as well. It is very useful for modifying values on the go, as an example.

#Getting started

Like usual when needing external resources, you first need to fetch the necessary files and load them up into your HTML document.
There's two ways of doing this: you can either download them from the github repository, https://github.com/Hallo89/Slider89, or fetch the files from the repository's website as seen below to always get the latest available version.
In both cases only two files are needed: slider89.js and slider89.css
You'll then need to link them in your HTML file like so:

<script src="yourpath/slider89.js"></script>
<link rel="stylesheet" href="yourpath/slider89.css">

or, fetching from the website:

<script src="https://hallo89.github.io/Slider89/slider89.js"></script>
<link rel="stylesheet" href="https://hallo89.github.io/Slider89/slider89.css">

Tip: I prefer to load both the script and style at the end of the document to squeeze an extra bit of speed out of the page because, given that the scripts are also at the end of the document which is usual practice, one will not need them before.

Now that everything is loaded up, we can start to create sliderz. All you have to know is the initialization method; everything else can be taken from the documentation section.
To create one instance of a slider, we do the following in our javascript:

var mySlider = new Slider89(targetElement);

This is pretty straightforward: we create a new instance of the Slider89 class with the new operator and pass the HTML-element we want the slider to be created in as paramter into the function.
Lastly, we obviously have to store the slider in a variable to be able to access it in future code (it is also valid not to use a variable, I'm doing this with the demos on this page for example, but this is a very rare case as you cannot do anything with it after initialization).
An all-default slider initialized plainly like this will look like the example you see above:

However, what would a framework be without means of configuration? To configure it we pass an additional parameter as object into the Slider89 constructor which we fill with configuration-properties. Here's an example: 

var mySlider = new Slider89(targetElement, {
  min: -50,
  max: 175,
  value: 80
});

#Documentation

This chapter will introduce all available methods, variables etc. with a thorough explanation

#Properties

These are the key to reading values from the slider and defining its properties.
Every property's value is accessed using <sliderInstance>.<property>

Configuration properties #

Define the slider's configuration as properties of an object passed as a parameter into various functions.

min
The minimum value of the slider's range
├ type: number
├ influencing: width at 'auto'
└ default: 0
max
The maximum value of the slider's range
├ type: number
├ influencing: width at 'auto'
└ default: 100
value
Used to specify the value the slider and its knob should start at
├ type: number
└ default: min
comma
How many digits the value should have after its decimal place
├ type: int
└ default: 0
width
The final width of the slider.
A number represents the width in pixels regardless of the slider's range, whereas the keyword 'auto' computes the width automatically from the slider's range in a 1:1 ratio.
If the property absWidth is false, which is default, the width is implicit; it is expanded by a small amount resembling the padding at the sides of the slider (The default padding is half of the slider knob's width either side) in order to get an expected result
├ type: number
├ unit: px
├ influences: absWidth, min, max
├ available keywords: 'auto'
└ default: 'auto'
caption
The caption you see below the slider
├ type: string
└ default: ''
classList
An array containing CSS classes to be added to the slider's HTML-structure
├ type: array<string>
├ influencing: HTML-structure
└ default: []
task
This value takes a function to be executed every time the slider is moved. The function must be declared without parenthesis, or be an inline anonymous function
├ type: function
└ default: undefined
taskMouseUp
This is the same as task, taking a function, which is, however, executed only every time the mouse button is released after the slider has been interacted with
├ type: function
└ default: undefined
absWidth
'Absolute width'; setting this to true computes the slider's width explicitly without taking the slider's padding into account
├ type: boolean
├ influencing: width
└ default: false
trimComma
If true, a value's digits after its decimal place are culled if they are 0. This is an aesthethik config property noticable at the value-displaying tooltip.
Example: 6.000 is trimmed to 6
├ type: boolean
└ default: true
tipDuration
A duration after how long the value tooltip should vanish when the slider hasn't been interacted with
├ type: number
├ unit: ms
└ default: 250
replaceNode
If set to true, the slider instance will replace the targetNode instead of being appended as a child inside it
├ WRITE ONLY
├ type: boolean
├ influencing: Slider89
└ default: false

Read-only #

The following properties are read only and may not be used as configuration properties

element
The HTML element (see HTML structure) of the final slider instance after initialization
├ READ ONLY
├ type: HTML element
└ default: undefined

Internal #

Below listed are properties which are internal to the framework and serve no purpose for the user but maybe you want to hack the framework idk but feel free to

taskLock
While dragging the slider, if the previous and new value are equal to each other, the specified task is executed once and taskLock is set to true, which prevents further executions of task if the value has still not changed. If the value has changed again, taskLock is set to false
├ INTERNAL
├ type: boolean
├ influences: executeSlider
└ default: false
tipTimer
The current timeout instance used for the value-tooltip which is terminated when the slider is dragged anew while a previous timeout instance was still running
├ INTERNAL
├ type: timeout
├ influences: executeSlider
└ default: undefined

#Methods (functions)

Constructor #

Slider89(<targetNode>[, <configProperties>])
The slider object constructor; initialize a slider instance by using the new keyword in front of it and binding it to a variable.
Example:
var myInstance = new Slider89(someElement);
or: 
var myInstance = new Slider89(someElement, {
  configProperty1: configPropertyValue1,
  configProperty2: configPropertyValue2
});
├ targetNode = HTML element the instance will be created in
├ configProperties = configuration properties
└ influences: replaceNode

Instance functions #

Functions under this category are called on a slider instance

<sliderInstance>.newValues(<configProperties>)
This function takes configuration properties and restructures the slider based on its new properties.
Although properties could be overriden using <sliderInstance>.<configProperty> = <newValue>, this function ensures every dependent and dependency is restructured as well.
Examples: it writes a new caption into the element; it sets a new width of the element; etc.
Additionally, if min or max is changed, the value recomputes itself based on the previous min-max-ratio
└ sliderInstance = An instance of a slider constructed via the Slider89 constructor

The slider89 object #

'slider89' (take notice that it's a lowercase "s"!) is a standalone object used for functions which are called without a reference to a slider instance

slider89.defaultValues(<configProperties>)
Called before a slider is created, it sets default values for every instance to apply
└ slider89 = standalone object

Internal #

As with the internal properties, these values serve no purpose for the user but are listed here to give the user (and hacker) full transparency

<sliderInstance>.buildElement(<targetNode>, replaceNode)
-> HTML element
Builds the slider's HTML structure into the specified targetNode, adding additional classes from the classList property, if present. If replaceNode is true, it replaces targetNode.
The function returns the created element to be written into a variable (-> element)
├ INTERNAL
├ influences: replaceNode, classList
└ influencing: HTML structure
<sliderInstance>.executeSlider(<clickedCoordinate>)
This function is executed every time the mousemove event, and more specifically, the mouseMove function is fired and handles the value, its tooltip and the specified task function
├ INTERNAL
├ clickedCoordinate = an int representing a screen-related x-coordinate where the mouse has been clicked
├ influences: element, task
└ influencing: value, tipTimer, taskLock
<sliderInstance>.checkTask(<task>)
-> <task> | false
A function evaluation whether a handed-over task is valid:
If it is, plainly return it without doing anything, if not, for example because it has been declared with parenthesis (which javascript cannot explicitly check), throw an error into the console and return false
├ INTERNAL
└ influencing: task / taskMouseUp
<sliderInstance>.computeWidth(<valueObject>)
-> int
If the valueObject's width is 'auto', compute the width automatically based on min and max, if it isn't, set the valueObject's width to its own width but take absWidth into account
├ INTERNAL
├ valueObject = an object preferably carrying values for width, min, max and absWidth
├ influences: min, max, absWidth
└ influencing: width
<sliderInstance>.mouseMove(event)
Is executed by the mousemove event, being its own function for the ability to remove the event listener afterwards, and executing executeSlider, parsing event.x
├ INTERNAL
├ event = The event parameter handed over by the event listener

#HTML structure 

<div class="slider89 [{additional classes}]">
  <div class="slider" style="width: {width}px">
    <span class="slider_knob" style="transform: translateX({value}px)"></span>
    <span class="slider_tooltip right hidden noselect">{value}</span>
  </div>
  <span class="slider_header">{caption}</span>
</div>
